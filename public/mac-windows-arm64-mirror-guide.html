M 系列 Mac 安装 Win11 ARM64

Plan F：第三方镜像源方案 — 零代理、零订阅的备用路径

📅 2026-04-26 🖥️ Apple Silicon · macOS Tahoe+ 📦 适用：VMware Fusion / UTM

核心思路

绕开 Microsoft 官方下载路径（被 Akamai 限速 + Esd2iso 不读 macOS 系统代理），从第三方拿到等价的安装介质（ISO 或 VHDX），再喂给 Fusion / UTM。本质是把"下载"和"安装"两步解耦。

背景：VMware Fusion 内置的"从 Microsoft 下载 Windows"向导在中国大陆 IP 下必败，因为 Esd2iso 工具进程不读 macOS 系统级 HTTP 代理设置。Plan F 不依赖 Fusion 的下载向导，因此对网络环境最宽容。

三类来源（按可信度排序）

1. UUP dump最推荐

  • 站点：uupdump.net
  • 原理：从 Microsoft Update 服务器拉原始 UUP 包，本地脚本拼装成 ISO；文件本身来自微软，只是分发链路不同

步骤：

  1. 浏览器访问 → 选 "Windows 11" → Architecture 选 arm64
  2. 选最新 Build（如 22631.x / 26100.x）→ 选语言（zh-cn 或 en-us）→ Edition（Pro 推荐）
  3. 下载一个 .zip，里面是 uup_download_macos.sh
  4. macOS 终端执行：
# 先装依赖
brew install aria2 cabextract wimlib

# 解压 + 跑脚本
unzip Windows11_ARM64_xxx.zip -d uup-build && cd uup-build
bash uup_download_macos.sh

等 30–60 分钟，得到 Windows11_ARM64_xxx.iso（约 5 GB）。

优点：能拿到最新预览/正式版、ISO 而非 VHDX（Fusion 装更标准）、社区背书强
缺点：拼装慢、依赖工具链；UUP 服务器在国内速度看 RP，必要时挂代理

2. next.itellyou.cn需自行核验

  • 老牌国内 MSDN 镜像站，历史上提供过 Win11 ARM64 VHDX/ESD
  • ⚠️ 不确定当前是否还有 ARM64 条目，最近一次大改版后部分版本下架；用前先去站内搜 "Windows 11" 并按架构筛选 ARM64
  • 文件分发走国内 CDN（磁力 / ed2k / 直链），下载快
  • 必做：下完后用 shasum -a 256 file.iso 对比站内公布的 SHA256，确认未被篡改

3. 论坛 / 网盘共享不推荐

  • 远景论坛、各类 Telegram 群、百度网盘共享
  • 风险：ISO 可能被植入后门 / 修改过 OEM 配置 / 装完弹广告
  • 用时必须对比官方公布的 SHA256（如果能找到）

安装流程

UUP dump ISO + Fusion

1. New VM → "Install from disc or image"
2. 选刚才生成的 .iso
3. 操作系统：选 "Windows 11 64-bit ARM"
4. 自定义硬件：CPU 4 核、内存 8 GB、磁盘 64 GB（动态）
5. 启动 → 走标准 Windows 安装向导

VHDX（官方 / itellyou）+ Fusion

1. New VM → "Create custom VM"
2. 选 "Microsoft Windows" → "Windows 11 64-bit ARM"
3. "Use an existing virtual disk" → 指向 .vhdx
   选 "Take this disk away from..."（避免共享导致的写冲突）
4. 启动即进 Windows OOBE

关键校验步骤（防坑）

# 1. 校验 SHA256（UUP dump 下载完会在终端输出预期 hash）
shasum -a 256 Windows11_ARM64_xxx.iso

# 2. 如果是从 itellyou 下的 VHDX，挂载检查内部
hdiutil attach -nomount file.vhdx
# 或用 7z 查看结构
7z l file.vhdx

# 3. 装完 Windows 后第一件事：
#    Settings → System → About
#    确认 Edition 和 Version 与下载时选择的一致，且无奇怪 OEM 标识
⚠️ 安全提醒：第三方镜像最大的风险不是装不上，而是装上了之后你不知道里面藏了什么。SHA256 校验是底线，能跳过的只有 UUP dump（因为是你自己从微软服务器现拼的）。

建议

如果只装 1–2 次，走 UUP dump：拼装慢但最干净、最可控，下来的 ISO 还能存着以后重装。
itellyou 仅作"UUP 服务器抽风时的备胎"，下完务必校验 SHA256。
