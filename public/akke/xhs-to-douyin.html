1这条链路在干什么

我们要把小红书上的视频笔记拿到抖音上发布，但抖音对原样搬运有相似度检测，文案风格也跟小红书不同。这条流水线把中间所有繁琐工序自动化：

① 解析小红书链接，拿到原视频
② ffmpeg 二改：水平翻转 + 音频变速 1.02x（破抖音 hash 检测）
③ LLM 重写文案：抖音风格的标题、标签、正文
④ 给 PM 一个下载链接 + 一键复制文案

整套端到端约 30 秒，PM 全程只用做两件事：贴链接、把结果搬到抖音 App。

💡
为什么不是直接搬运？ 抖音对原视频 + 原音频会做 hash 指纹检测，相似度过高会限流。水平翻转改变画面 hash，音频变速 1.02x 改变音频指纹 —— 这两步的视听代价几乎为零（人眼/人耳几乎察觉不到），但能显著降低被识别为搬运的概率。

第一次打开网址，在底部"访问 token"输入框里粘一次，浏览器会记住，下次免填。

🔐
token 是简单访问控制，不是机密。 不要把 token 贴到任何外部对话框（公开 Slack/微信群、社交媒体、第三方 AI 工具）。只在 Akke 内部使用，外传等于送给竞品。
⏳
服务空闲会休眠（节省成本，正常行为）。第一次访问可能要 5-10 秒冷启动，之后每次秒开。如果发现 "Connection refused" 多刷一次就好。

3三步操作（PM 主流程）

第 1 步

从小红书 App 复制笔记链接

在小红书 App 里打开目标视频笔记 → 点右上角"···" → 选"复制链接"。系统剪贴板会得到类似这段（整段不用剪，系统自动抽 URL）：

140 装成200 ㎡｜极简奶油风果然显大又高级❗️ 我家... http://xhslink.com/o/1e7HQQPU9FD
⚠️
只支持视频笔记。图文笔记跑会失败，会有错误提示。如何分辨：视频笔记封面右上角有播放图标。
第 2 步

打开网址 → 粘贴 → 点"生成抖音版本"

打开 https://xhs-pipe.fly.dev/，整段粘到顶部输入框，点按钮。页面会显示状态条：

🔄 正在解析链接、下载视频、ffmpeg 二改、LLM 重写… 通常需要 10-30 秒。

不要切换浏览器标签，等结果出来。

第 3 步

拿三样东西 → 上传抖音 App

  • 二改视频 —— 页面里直接预览，点 "↓ 下载二改视频" 存到电脑/手机
  • 抖音文案 —— 标题 / 标签 / 正文 各一格，每格右上角有"复制"按钮；最下方有"一键复制全部"
  • 原小红书对照 —— 下半屏摆着原标题/描述/标签，方便人工校对 LLM 改得对不对

在手机抖音 App 里 → 上传刚才下载的视频 → 在标题/正文框分别粘对应内容 → 发布。

5v0 二改做了什么

操作作用视听代价
水平翻转破画面 hash 检测人眼几乎察觉不到（除非画面里有文字）
音频变速 1.02x破音频指纹检测视频时长缩短 ~2%，人耳几乎察觉不到

视频里如果有大段文字（如品牌 logo、屏幕截图、字幕中文）翻转后可能反向阅读 —— 这种笔记建议手动校对再发，或者跳过。

⚠️
v0 没做的事：BGM 替换、品牌水印、上下裁切、变焦。这意味着原 BGM 还在 —— 如果原视频用的是小红书热门 BGM，到抖音可能仍被识别。下一版会加 BGM 替换。

6故障排查

症状原因修法
HTTP 404 / "note deleted"链接太老 / 笔记被删 / 链接格式不对换条新链接（7-14 天内发布的最稳）
HTTP 401 "invalid app token"token 没填或填错检查底部 token 输入框是 xhs-pipe-0e993127
"not a xiaohongshu URL"粘的不是小红书链接从小红书 App "复制链接"重粘，别从浏览器地址栏拷
"login wall? geo-blocked?"笔记需要登录看 / 服务出口被临时封找 Owner 上报，多半要重启或换区
处理 60 秒还没回视频特别大 / 服务冷启动 + 长视频多等 30 秒；超过 90 秒再换链接
视频网页里能播但下载失败浏览器拦截 / 网络中断右键链接"另存为"
LLM 标题感觉不对偶发：原笔记信息量太低 / LLM 创意飘同一条再跑一次（会出不同结果）；或手动改

遇到表里没列的错误，把红色错误条整段贴给 Owner，多半是某种异常笔记格式没覆盖到，加个 case 就能修。

7注意事项与红线

  • 链接 TTL：小红书短链发布 7-14 天后可能 404，越新越稳
  • 只支持视频笔记，图文笔记下一版再说
  • 同一条链接重复跑会生成新的视频文件，旧文件保留在服务器（1GB volume 够装几百条）
  • token 不外传，不要贴公开聊天 / 不要发给客户
  • 抖音文案 LLM 出的是建议，建议人工 quick scan 一遍 —— 偶尔会有事实性偏差（如把"奶油风"写成"奶咖风"）
  • 不要批量灌入抖音，单号一天发 ≤ 3 条二改内容观察效果，没限流再加量
🚫
红线 · 不要做：
  • 不要发带原作者明显水印（左上角 ID、自带签名）的视频 —— 这种侵权风险高，建议跳过
  • 不要发带明显小红书 UI 元素的截屏 —— 抖音对竞品引流容忍度低
  • 不要把同一条二改视频投到多个抖音号 —— 抖音跨号 hash 检测一样敏感
  • 不要在客户账号没经 Owner 确认的情况下批量上线 —— 账号是客户资产

8后续会补

v0 是单链接交互版，先用一周看 PM 实际痛点。已规划但还没做的：

  • BGM 替换 —— 准备 10-30 首抖音商用 BGM 素材库，自动随机替换原音轨
  • 品牌水印 —— 右下角加客户品牌 logo（每个客户传一次素材）
  • 批量录入 —— Lark Base 表 + n8n 编排，PM 一次粘 20 条链接自动跑批，结果落"草稿池"
  • 抖音直发 —— 暂不做（账号风控风险高，人工最后一道审核更稳）
  • 图文笔记支持 —— 抽图 + Wan2.2 I2V 生视频 + TTS 配音 —— 工程量大一档，先看视频笔记够不够用
💬
有反馈直接找 Owner。看 LLM 出的某条标题特别好 / 特别糟、想加某种新二改、跑出了奇怪错误 —— 直接截屏发 Lark，所有反馈都会变成下一版的输入。
