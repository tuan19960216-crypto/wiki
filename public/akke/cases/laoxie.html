破冰文案 V3 · 四段式定型
这四段是 Akke ice_break 阶段所有破冰文案的标准结构 — 不结束在客气话，而是结束在问题上。本条 DM 在评论是「直接问价」的情形下，第三段顺势把"千把一平投影算"摆了出来 — 这一选择会决定客户后续是把对话往「报方案」走、还是把数字带去外部比价。当下处于「等首次回应」窗口，结果仍未定。

复盘 · 这条线索教会我们的

5 个可复用的系统观察

1
「什么价位」是 product type 的标准入场券。老谢这一句直接把 LLM 标签锁定到 product · medium · active 三轴 78 分 — 评分一过 70，conversation 被自动建出来、进 ice_break 流水线。这是从评论到对话的关键 gate。
2
IP 属地是免费的地域钩子。「陕西最近问全屋柜体的多」— 数据本就在评论 API 里，不用反问、不用客户额外披露。在客户没自报市的情况下，省级回退是 ice_break 标配。
3
四段式破冰把"招呼 / 视频共鸣 / 业务地域 / 反问"压缩在一条 DM 里。不结束在客气话，而是结束在问题上。本案第三段处理了客户的价格疑问、第四段抛出「板材 / 五金」二选一探测钩 — 客户是否买账由下一回合验证。
4
触达延迟 3 小时 27 分 = 端到端流水线时延。评论 18:06 留下、DM 21:33 推过去 — 这是抓评论 → LLM 评分 → 起草 → 真机推送整条线路的端到端可观测延迟，落在 active 段（<24h）。
5
stage 真实卡在哪显示哪。ice_break 没继续推进就显示 ice_break，sec_uid + sent_at + comment_id 都可追溯 — 案例库不为"完整结局"而隐藏当下真实状态。下一次轮询拉到客户回复，本页 ACT 05 自然展开。
