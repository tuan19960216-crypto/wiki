5 个可复用的系统观察

1
行业词是用户身份的免费披露。「多层板」是装修业内的板材类型词 — 一句话让 LLM 把 keyword 锁在 product / 板材类，进入 ice_break 流水线。同样原理适用于「颗粒板 / 生态板 / 欧松板 / 兔宝宝 / 莫干山」等任何专业术语。
2
工厂号 + 同省 IP = 软同城候选。本案号源在南宁、客户在广西 — 严格同城规则因 comments.city = NULL 不触发，但实质上同省工厂号下面的评论极大概率落在号源服务半径内，这是 region prompt 可继续优化的方向。
3
四段式破冰把"招呼 / 报价 / 紧迫 / 反问"压缩在一条 DM 里。面对"什么价"型评论的标准结构。本条 DM 的"七百起每平"+ "档期要提前约"是工厂号场景下的常见组合 — 是否激活客户后续动作要由下一回合验证。
4
触达延迟 5 小时 26 分 = 端到端流水线时延。评论 08:27 留下、DM 13:53 推过去 — 这是抓评论 → LLM 评分 → 起草 → 真机推送整条线路的端到端可观测延迟，落在 active 段（<24h）。
5
stage 真实卡在哪显示哪。ice_break 没继续推进就显示 ice_break，sec_uid + sent_at + comment_id 都可追溯 — 案例库不为"完整结局"而隐藏当下真实状态。10 天窗口下，本条线索的训练数据价值正在累积。
