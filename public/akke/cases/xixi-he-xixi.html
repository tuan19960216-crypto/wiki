没反问的破冰 · 是模板变体还是疏漏？
从结构上看：本案破冰 3 段 vs 标准四段式（招呼+视频复述 / 地域接业务 / 工艺/价格 / 反问）少了"反问"。可能两种解读：①运营手动给 prompt 加了"对门窗品类不强反问"的指令，因为门窗决策周期长、客户多需要慢慢比较；②V3 prompt 在跨品类（柜体 → 门窗）场景下结构稍变形 — 没有专门的门窗 opener 模板。

事后看效果不差：客户回了「好的，谢谢～」 — 礼貌承接，没有抵触。可能没反问反而让她不感觉被推着回答。反问是降回答门槛的工具，但不是必须 — 对礼貌型客户，陈述式结尾给的"思考空间"也是种降阻力。

6 个 nurture 阶段 + WDA 通道 + 手工跟进的原则

1
「好的，谢谢～」短回复也是合格 nurture 触发信号 · 但不是承诺。客户主动响应过 = 对话已双向打开 = 应该进 nurture。但礼貌承接 ≠ 高购买意愿 — 本案 nurture 跟进 #3 后客户已沉默 ≥ 21 小时。不要因为短回复就当作冷淡 · 也不要因为升级 nurture 就当作"快成单"。stage 是流水线状态，不是购买预测。
2
nurture 跟进的关键 = 提供新信息密度 · 不是重复破冰。本案 #3 把"450 预算 / 系统门窗"具体化到「1.4mm vs 1.0mm 玻璃料厚」「5+12+5 vs 4+9+4 中空层」「同价隔音差一倍」 — 这些是装修圈内人才知道的被偷工减料细节，客户读到立刻意识到对面真懂行。nurture 是"再升一档专业度"的窗口，不是"换种方式重复破冰"。如果跟进还在用 #1 已讲过的卖点 → 客户感觉绕回原点，不会再回。
3
手工跟进可以塞 V3 不敢编的"行业内幕"数据。「1.4mm vs 1.0mm 玻璃料厚」「5+12+5 vs 4+9+4 中空层结构」「同价隔音差一倍」 — V3 prompt 一般不主动塞这种具体行业规格数字，因为 LLM 可能编错（怕"5+12+5"被胡乱改成"6+12+6"）。手工 / Claude 协助跟进的独特优势是：运营在场可以塞真实行业数据 + 校验数字准确。同大欣 case 塞具体设计师真名「武海艳」的逻辑。
4
「撤回 = 重写」是运营深度参与对话的高确定性证据。本案 5/20 13:56 客户端显示"你撤回了一条消息" — 推断运营/AI 发了初版跟进 ≤2 分钟内立刻撤回重写。自动 followup pipeline 一般不撤回（pipeline 算法不会"刚发完就反悔"）。撤回信号说明运营在 review 跟进文案质量 + 不满意立刻替换。这是手工运营独有的质量控制动作 · 也是抖音 IM 的产品特性给了运营 2 分钟反悔窗口。但抖音的撤回事件 inbox 同步链路不会捕获 → DB 完全无痕，监控面板看不到这种运营动作。
5
V4 评分对"含数字 + 询问语气"组合识别准 · 比纯关键词强一档。本案评论「可以帮忙看下吗？450」没匹配任何关键词词典，但 LLM 从"450"+ 询问语气拿到 78 分 product。这是 V1 老评分（家怡526「想订」漏判到 50 分）的升级。后续 prompt 优化方向：含具体数字（金额/尺寸/平米）+ 询问标点应该自动加权，让评分更稳。
6
WDA silent_drop_suspected 检测有假阳性 · 需要自动回填校正。本案 DM 实际送达了但被标为 suspected silent_drop，运营人工改了 error_message。silent_drop 检测的阈值/条件需要调优，更重要的是当 DM 收到客户回复时应该自动反推之前的标记为假阳性，否则会污染监控数据 + 触发不必要的重试。同空间向量 messaging_account_id NULL 是同一类数据完整性自动修复问题。
