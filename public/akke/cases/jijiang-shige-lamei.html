4 个字也能拿 78 分 · 警惕的客户也能拿下

在灿哥聊装修（号源池里的一个本地号 · 服务市未标）发的那条「5 月开工 25 步流程图」视频底下，她打了 4 个字（原评论）— 案例库目前最短评论。同时她的抖音签名是「少来沾边滚」 — 防御性极强的姿态。其他案例的客户多半是默认昵称 / 友好昵称，本案是明确表达过"别来烦我"的客户。

3 件事在这条 4 字评论 + 签名组合里同时暴露：①「需要攻略」= 索要资料，product type 信号里最直接的转化前置动作；②评论时间 22:20 = 晚上刷视频，工作日睡前"明天开始查装修"的心态；③签名「少来沾边滚」+ 真实头像 + 实人 profile = 警惕型实人客户（不是默认 ID，不是 bot，是有自我表达的真实用户）。

极简评论的判断逻辑 + 防御性签名的解读：评论字数越短，越要看动词的转化距离。「需要攻略」里的动词「需要」+ 名词「攻略」组合 = 转化漏斗最上游的"我要开始研究了"。但本案叠加了一层防御性签名，意味着她平时收到大量垃圾私信、对陌生联系警惕度高。「需要攻略」对她来说很可能是对视频内容的反馈，不是邀请陌生 DM。AI 破冰需要对得起她的警惕：不能套话、不能群发感、必须证明读过她的具体上下文。事后看（见 #2），她接受了聊天 + 给了户型 — 说明 AI 这一关过了。

5 个评论解读 + 跟进决策的原则

1
防御性签名 ≠ 拒绝触达 · 客户 profile 是更全的判断依据。本案客户签名「少来沾边滚」 — 看起来是"别 DM 我"的明显信号。但她仍然接受了聊天、给了户型。签名只是她平时的防御态，不是这条具体评论的姿态。判断要不要触达，看的是 ①评论 type（产品型 / 价格型 / 索要型）+ ②评论的承接性动词强度 + ③视频上下文，不是看签名。但破冰话术要对得起防御性签名：拒绝套话、不能群发感、必须证明读过她的具体上下文。本案 AI 引用"流程图第 18 步"做到了。
2
视频 metadata 完整时，破冰要"引用视频里的具体序号 / 关键节点"。本案破冰直接引用「流程图里柜体定制排第 18 步对吧」— 比泛泛「我看了你那条视频」具体一个量级。客户读到这句会立刻意识到"这个人真看过、还记得里面具体内容"。事后看，这是穿透「少来沾边滚」防御态的关键句。判断标准：能不能从视频 title / description / hashtag 里抽出一个客户也记得的细节（具体步骤序号 / 关键参数 / 工艺名词）。如果可以 — 必须用。
3
跟进文案"复述户型"自动修正破冰称呼的小瑕疵。破冰用了「即将是个辣妹姐」拗口称呼 — 但跟进 #3 没再重复，开头改成「3室一厅好规划」（直接复述户型 + 评价）。这是 V3 prompt 的自我修正：当客户给了具体参数（户型/面积/材料），跟进 DM 自动用参数当称呼上下文。未来 prompt 可以更主动：让 V3 优先用客户给的具体参数当跟进 DM 的起手段，降低"称呼"在称呼上的权重。
4
客户给了户型 → 首次跟进就可以报价 · 别再藏数字。其他案例的第一轮跟进都没给具体价格（用户3824342698628 给工艺数字、空间向量提"省 30%"但不报数）。本案 AI 直接给「活动3w就能落地」 — 大胆策略。合理性：①客户已经给了户型，3w 是有依据的具体数字不是空中报价；②"需要攻略"的潜台词包含"我要预算参考"；③她已经过了"信任试探"阶段。判断时机：客户给参数（户型 / 面积 / 材料 / 尺寸）那一刻起，AI 跟进可以直接报价 — 客户主动给参数是邀请报价的最强信号。
5
加微钩子的"职位词"应当匹配交付物专业度。本案用「让设计师把剖面和明细报价」替代其他案例的「让我同事发你参考」。"设计师" > "同事" 的专业感来自：①"设计师"是装修客户认知里的专业角色（建筑师/医生类的高门槛职位词）；②"剖面 + 明细报价" = 设计公司级交付物，3w 报价配设计师交付，专业服务感成立；③"同事"是泛公司角色，匹配的是"发个目录看一下"的低专业度交付。用职位词当心理价签，配合报价数字一起立专业人设。
