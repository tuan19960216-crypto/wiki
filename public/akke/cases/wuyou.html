ACT 05 · 价格信号偏软 本案"拉清单" vs 大欣"报价" vs 天涯客"报价清单"

"拉清单" — 一个字之差，案例钩子 vs 报价钩子

本案 DM 的钩子用的是 "具体方案私信给你拉清单？"——同红尘有梦7865案的钩子一模一样。"拉清单"只承诺了一个动作，不承诺清单里有什么。这是本案文案的最大短板，对比大欣和天涯客的钩子，差距明显。

⚠️ 三案价格钩子对比
无忧（本案）："具体方案私信给你拉清单"
• "方案" — 模糊 · "清单" — 动作，没价值 · 客户读完不知道加 v 能拿到什么

大欣案："加 v 给你出全屋柜体免费设计方案 + 报价"
• "免费" — 价值锚 · "设计方案" — 内容 · "+ 报价" — 价格承诺 · 客户知道能拿到三件东西

天涯客案："加 v 发你同户型实景案例和详细报价清单"
• "同户型实景案例" — 佐证 · "详细报价清单" — 价格明细 · 客户知道能拿到佐证 + 报价
🔍 关键差异 · 一个字
"拉清单"（本案） 和 "报价清单"（天涯客案） 只差两个字，但承诺密度完全不同：
• "拉" = 动作（我把清单从某处拉出来给你）
• "报价" = 价值承诺（清单里有具体价格数字）

客户看到"报价清单"会想"我能比价 / 我能决定"；看到"拉清单"会想"什么清单？"。
高意向 + product + high specificity 的客户已经在做决策——他们要的是能决策的信息（具体数字、案例、对比），不是"私信再聊"这种延后型动作。
为什么 93 分 + 2.37h + 视觉细节复述对了 — 还是 0 回复
可能解释：客户已经在做决策（看款 + 量好尺寸 + 来问适配），她要的是数字。本案 DM 给了她：视觉细节复述 ✓ / 2.1m 适配确认 ✓ / 板材推荐 ✓ / 反问平层复式 ✓ — 但没给报价、没给案例链接、没给"加 v 能拿到什么"。
她可能读完想"懂了，但你没说价格，我还得问"——而问的成本（加微信 + 备注暗号）远高于直接划走去问下一家。
结论：93 高分 + 闪电响应 + 视觉细节复述 = 必要但不充分。价格信号软化 = 临门一脚的临门一脚。
ACT 06 · status=skipped 的 funnel 异常 comments.status = skipped · 但 dm-phone-sender 仍发了

funnel 漏洞：旁路通道绕过了 funnel 状态机

本案的 comments.status 写的是 "skipped"——意思是 analyze cron 在打完 93 分后，因某条 dedupe 或 cid 解析规则把它从标准 funnel 中排除了。理论上 status=skipped 的评论不应该收到 DM；但本案的 DM 是被 dm-phone-sender 这个旁路工具独立挑出来发的，绕过了 status 检查。

🐞 funnel 状态机的双向漏洞
漏洞 ①（上行）：评论被 funnel 打成 status=skipped，但 dm-phone-sender 不读这个字段就发了——funnel 的"跳过"判定不被旁路工具尊重。
漏洞 ②（下行）：DM 发出后没回写 conversations / messages 表——funnel 不知道这条线索其实已经触达过客户。
双向漏洞结合的后果：本案在 funnel 系统看来是"被跳过的、从未触达的线索"；在客户那里却是"已经收到 1 条 DM 的活线索"。这两个状态同时存在。修：dm-phone-sender 必须 ① 读取 status 决定是否发；② 发送成功后回写 funnel 状态。

5 个必须记住的系统结论

1
2.37h 是 funnel 能做到的时效上限。本案两段延迟（入库 1.23h + DM 1.13h）证明：当 scraper 跟新视频高优先策略 + 运营在线 + 评论时间不踩雷——funnel 能跑到 2-3h SLA 之内。这是工程层面的"理想样本"，不是物理极限。
2
93 高分 ≠ 自动转化。本案是 funnel 最理想画像（product · high · active · 93 分），时效达标，但因价格信号软依然 0 回复。评分是入门券，钩子的价值密度才是临门一脚。修：把"价格钩子成色"纳入 DM 生成的硬约束，禁用"拉清单 / 私信聊 / 看一下"这种延后型动词。
3
"拉清单" vs "报价清单" — 一个字之差。这是本案最重要的文案教训。"拉" = 动作；"报价" = 价值承诺。高意向客户已经在决策，要的是能决策的信息。修：DM 的钩子文案必须出现具体的价值名词（报价 / 案例 / 方案 / 户型图），不能只出现动词。
4
视觉细节复述是 product · high specificity 的必备开场。本案用"奶咖配色 + 现代简约 + ENF 板材"三个视频里的关键词，让客户感觉"看了视频的不是模板群发"。这是 prompt 应该固化的视觉细节抓取规则：从视频 title / cover OCR / hashtag 里抽出 2-3 个视觉关键词喂给 LLM。
5
status=skipped + DM 已发 = funnel 双向漏洞。本案的 comments.status="skipped" 但 dm-phone-sender 仍发了 DM——这是旁路通道绕过 funnel 状态机。修：① dm-phone-sender 必须读 status 决定发不发；② 发送成功必须回写 conversation。否则 funnel 系统永远看不到这条已触达的线索。
