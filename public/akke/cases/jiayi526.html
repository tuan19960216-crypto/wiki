评分 50 · 但她明明说"想订"

Akke 的分析 cron 把评论喂给 LLM 评分。模型给了 50 分中意向，comment_type=null / specificity=null / freshness=null — 这是 V1 评分链路打的分（V4 评分体系于 2026-05 推出，V1 的评论拿不到三轴标签）。50 分跨"高意向"门槛（一般 60+）但漏算"想订"这种明确购买动词。这是案例库最有教学价值的"评分漏判"案例。

5 个购买动词漏判 + 跨省话术的原则

1
「想订」是强购买信号，但 V1 评分模型未必能直接反映。家怡526 评论里"想订"两个字 = 销售距离最近的状态之一，可 intent_score 只给到 50 分中意向。comment_type = null 说明这是 V1 评分链路打的分；新一轮 comment_type 升级后，含"想订/想要/求购/求介绍"这类购买动词的评论应自动加权到 product · high。
2
评论里的数字规格 → 必须在 opener 里接住。客户说"3.6 长度"+"几个脚"，AI 立刻接"3.6 米跨度做 3 个粗脚稳当" — 这是 ice_break 阶段建立专业感的最低成本操作：用客户给的具体规格反哺技术建议，比空泛"我们能做"有效一个数量级。
3
软定价 = 用变量分解代替硬数字。「价格看你皮质+木种」是本案 opener 最干净的一行 — 既回应了"价格是多少呢"的疑问、又没给一个可比价的数字锚。价格留到第二轮谈，不留可比价数字是 ice_break 阶段定价话术的核心模式。
4
跨城软同城 = 双层回退。号源在成都、客户在广东，硬同城（is_same_city=false）不触发；AI 切到客户视角的近场，用「广东订得多」省级覆盖 + 「你那边精装房多」省级 micro-context 双层叠加 — 这是 region prompt 处理跨城场景的标准答案。
5
触达时效是 ice_break 转化的独立变量。本案 opener 话术几乎是当前 prompt 能写到的最好水准，但 7h 触达可能错过了"想订"信号的购买冲动窗口 — 话术 ≠ 转化保证。"想订/想要"类强信号评论应触发快速通道，绕过常规评分 cron 节奏（同空间向量「怎么联系」类的紧迫度教训）。
