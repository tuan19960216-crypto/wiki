复盘 · 这条流量教会我们的

4 个对定价 / 商品矩阵的重要发现

1
「不订阅但付费」是真实细分，227⭐ 全部花在 store 而非 plan。Vivi 应该把 gem 包定位为「单次承诺产品」而不是「订阅备胎」。99 / 199 / 499 三档 gem 包对她已经验证有效。
2
付费意愿 ≠ 订阅意愿。她的 plan_subscribe 3 次 / plan_purchase_complete 0 次完整说明：订阅文案在她身上不工作，但 store 文案工作。两条漏斗应该按用户人群分别优化，而不是合并 KPI。
3
11 次 paywall 显示是高消耗。若每次 paywall_shown 都展示「订阅 + gem 包」两个选项，且她每次都选 gem 包，应该把 gem 包推到更靠前的位置 — 而不是反复推订阅。
4
image 失败 3/5 必须查。这种用户对体验断点最敏感。下一次 daily_monitor 应该把单用户 image 失败率列为告警维度。
📋 会议讨论清单
  • 商品文案 A/B：把订阅 paywall 改成「gem 包优先 + 订阅次选」，跑两周看转化变化。
  • image 失败率告警：连续 3 张失败的用户，应该自动触发客服 outreach 或赠送 5💎。
  • 「不订阅但付费」用户画像：跑一遍全表，找出所有 stars_paid > 0 但 sub_type = free 的用户，看是否能形成独立人群。
