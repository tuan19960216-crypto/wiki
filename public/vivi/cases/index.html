这个案例库给谁看？给 PM、运营、内容团队，还有所有需要从「指标聚合」回到「真实用户」的人。 每一条 paywall_shown、每一次 invoice_callback、每一份 affinity 都对应一个具体的 Telegram 用户。 我们这里隐去用户 ID 后五位之外的部分（如 tg_36927），其他数据都是 Turso 数据库里的真实记录。

讨论方式： 每个案例独立 URL，会议上可以单独打开来看。每个案例末尾有「📋 会议讨论清单」 — 这是我们建议团队优先讨论的产品/工程动作。

下次怎么追加？从 Turso 的 users / transactions / events 表跑相同的 join，挑「单用户高 paywall_shown / 高 invoice_callback / 异常 chat_reset」三类信号，每周补 3-5 个新案例。
